## 일목균형표란?

일목균형표(一目均衡表)는 1960년대 일본의 기자 호소다 고이치가 개발한 지표입니다. "한 눈에 균형을 볼 수 있는 차트"라는 뜻으로, 지지, 저항, 추세 방향, 모멘텀을 한 화면에서 보여줍니다.

대부분의 지표가 하나만 측정하는 것과 달리, 일목균형표는 5개의 선과 음영 처리된 "구름" 영역을 제공합니다:

```
1. 전환선 (Tenkan-sen)  = (9봉 최고가 + 9봉 최저가) / 2
2. 기준선 (Kijun-sen)   = (26봉 최고가 + 26봉 최저가) / 2
3. 선행스팬 A           = (전환선 + 기준선) / 2, 26봉 앞에 표시
4. 선행스팬 B           = (52봉 최고가 + 52봉 최저가) / 2, 26봉 앞에 표시
5. 후행스팬 (Chikou)    = 종가를 26봉 뒤에 표시
```

선행스팬 A와 B 사이의 영역이 "구름"(Kumo)을 형성합니다.

## 읽는 방법

### 구름 (Kumo)
- **가격이 구름 위** → 상승 추세
- **가격이 구름 아래** → 하락 추세
- **가격이 구름 안** → 명확한 추세 없음, 거래 회피
- **두꺼운 구름** → 강한 지지/저항
- **얇은 구름** → 약한 지지/저항, 돌파 가능성 높은 구간

### 핵심 시그널
| 시그널 | 조건 | 강도 |
|--------|------|------|
| **골든 크로스** | 전환선이 기준선 위로 교차 | 구름 위에서 발생 시 강력 |
| **데드 크로스** | 전환선이 기준선 아래로 교차 | 구름 아래에서 발생 시 강력 |
| **구름 돌파** | 가격이 구름 위/아래로 돌파 | 추세 전환 신호 |
| **구름 꼬임** | 선행스팬 A와 B 교차 | 미래 추세 전환 예고 |
| **후행스팬 확인** | 후행스팬이 26봉 전 가격 위/아래 | 모멘텀 확인 |

### "5선 확인" 최강 시그널
모든 요소가 동의할 때 가장 강력합니다:
1. 가격이 구름 위
2. 구름이 녹색 (스팬 A > 스팬 B)
3. 전환선 > 기준선
4. 후행스팬이 가격 위
5. 미래 구름이 녹색

실전에서 5개 모두 일치하는 경우는 드물고, 모두 일치할 때쯤이면 이미 움직임의 대부분이 끝났습니다.

## 크립토에서 일목균형표가 어려운 이유

일목균형표는 **1960년대 일본 주식의 일봉 차트** 용으로 설계되었습니다. 기본 파라미터(9, 26, 52)는:
- 9 = 1.5 거래주
- 26 = 1 거래월
- 52 = 2 거래월

### 크립토의 문제점
1. **24/7 시장**: 원래 파라미터는 주 6일 거래를 가정. 크립토는 쉬지 않음
2. **극단적 변동성**: 구름이 끊임없이 교차되어 거짓 신호 발생
3. **알트코인 잡음**: BTC/ETH에서는 나은 편이나 소형 알트에선 거의 무용
4. **지연**: 26봉 이동은 1시간 차트에서 하루 이상 전 데이터를 보는 것
5. **시각적 복잡성**: 5개 선 + 구름 = 정보 과부하. 보고 싶은 것만 보게 됨

### 백테스트 현실
535개 알트코인, 2년+ 데이터(1시간 봉):
- 순수 일목 시그널(TK 크로스 + 구름 방향)은 **수익 팩터 1.0 미만**
- 횡보 시장에서 너무 많은 시그널 (크립토는 ~60% 시간을 횡보)
- 지연으로 인해 진입이 일관되게 늦음

## 일목균형표가 유용한 경우

한계에도 불구하고 특정 요소는 잘 작동합니다:

- **기준선을 동적 지지/저항으로**: 단순 이동평균보다 반등 레벨 식별에 우수
- **구름을 추세 필터로**: "구름 아래에서만 숏" 은 유효한 필터
- **구름 꼬임으로 레짐 감지**: 미래 구름 색상 변화 시 레짐 전환 가능성
- **상위 시간대**: 4시간, 일봉에서 잡음이 크게 감소

## 크립토용 조정 파라미터

24/7 시장에 맞게 기본값을 수정하는 트레이더들:

```
전통적:    9, 26, 52
크립토 24/7: 10, 30, 60 (주말 없음 반영)
공격적:    7, 22, 44  (빠른 시그널)
```

"올바른" 크립토 일목 설정에 대한 합의는 없습니다. 모든 수정은 반드시 백테스트로 검증해야 합니다.
