# PRUVIQ Service Plan v1.0

> 크립토 전략 시뮬레이션 + 시장 컨텍스트 플랫폼
> 작성일: 2026-02-15

---

## 1. 서비스 정의

### 1.1 한 줄 정의
**PRUVIQ는 "투명한 오픈소스 엔진으로 크립토 전략을 검증하고, 시장 맥락 속에서 인사이트를 발견하는 플랫폼"이다.**

### 1.2 핵심 가치 제안 (Value Proposition)

#### 문제 (Problem)
- 기존 백테스트 플랫폼: 블랙박스 엔진으로 신뢰성 의심
- 과적합 위험: 과거 데이터 최적화만으로 실전 실패
- 단편적 정보: 전략 성과와 시장 맥락이 분리됨
- 높은 진입장벽: 코딩 필수 또는 복잡한 UI

#### 해결책 (Solution)
- **투명성**: 오픈소스 엔진(autotrader 기반)으로 로직 공개
- **검증성**: OOS(Out-of-Sample) 테스트 + Monte Carlo 시뮬레이션
- **맥락 통합**: 전략 성과를 시장 이벤트/뉴스와 함께 분석
- **접근성**: 코드 없이 파라미터 조정, 직관적 대시보드

#### 차별점 (Differentiation)
| 플랫폼 | 오픈소스 | OOS 검증 | 시장 맥락 | 코딩 불필요 | 크립토 전문 |
|--------|---------|----------|----------|------------|------------|
| PRUVIQ | ✅ | ✅ | ✅ | ✅ | ✅ |
| QuantConnect | ✅ | ❌ | ❌ | ❌ | 부분 |
| TradingView | ❌ | ❌ | ❌ | ✅ | 부분 |
| 3Commas | ❌ | ❌ | ❌ | ✅ | ✅ |

### 1.3 타겟 페르소나

#### Persona 1: 초보 트레이더 (크립토 입문 6개월 이내)
- **니즈**: 검증된 전략으로 안전하게 시작, 기본 개념 학습
- **Pain Point**: 무엇을 믿어야 할지 모름, 큰 손실 두려움
- **PRUVIQ 활용**:
  - 전략 카탈로그에서 검증된 전략 탐색
  - 시뮬레이션 결과로 리스크/수익 이해
  - 시장 컨텍스트로 "왜 이 시기에 잘/안 됐는지" 학습
- **성공 지표**: 첫 전략 선택 후 3개월 생존율

#### Persona 2: 중급 트레이더 (경험 1-2년, 월 거래액 $1K-10K)
- **니즈**: 기존 전략 개선, 새 아이디어 백테스트, 과적합 확인
- **Pain Point**: 백테스트는 좋은데 실전 실패, 파라미터 최적화 어려움
- **PRUVIQ 활용**:
  - 파라미터 커스텀으로 자신만의 변형 전략 테스트
  - OOS 검증으로 과적합 여부 판단
  - 멀티 거래소 비교로 슬리피지 최소화
- **성공 지표**: 월 1회 이상 시뮬레이션, Pro 전환율 35%

#### Persona 3: 고급 트레이더/퀀트 (경험 3년+, 월 거래액 $10K+)
- **니즈**: 복잡한 포트폴리오 시뮬레이션, 거래소 연동, API 활용
- **Pain Point**: 기존 플랫폼은 단일 전략만 지원, 데이터 접근 제한
- **PRUVIQ 활용**:
  - 멀티 전략 포트폴리오 상관관계 분석
  - 거래소 API 연동으로 실제 포지션과 비교
  - 오픈소스 엔진 코드 리뷰/기여
- **성공 지표**: Pro 플랜 유지율 80%, API 사용률

### 1.4 포지셔닝 맵

```
                고급 기능
                    ↑
        QuantConnect │ PRUVIQ (Pro)
                     │
블랙박스 ←──────────┼──────────→ 투명성
                     │
         3Commas     │ PRUVIQ (Free)
                     │
                    ↓
                단순 기능
```

- **X축 (투명성)**: 블랙박스(좌) ↔ 오픈소스(우)
- **Y축 (기능)**: 단순(하) ↔ 고급(상)
- **PRUVIQ Free**: 우하(투명+단순) - TradingView 대체
- **PRUVIQ Pro**: 우상(투명+고급) - QuantConnect 대체

## 2. 전체 서비스 플로우 (User Journey)

### 2.1 Journey Map Overview

```
┌─────────────────────────────────────────────────────────────────┐
│ Discovery → Exploration → Experimentation → Conversion → Habit │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 상세 단계별 플로우

#### Stage 1: Discovery (발견)
**목표**: 3분 내 핵심 가치 이해

```
[SNS/검색] → [랜딩페이지]
                ↓
         "3초 시선 확보"
         - Hero: "BB Squeeze 전략, 2년 실적 +158%"
         - CTA: "가입 없이 바로 체험"
                ↓
         [데모 대시보드 진입]
         - 실제 전략 결과 (535코인, 2년)
         - 코인별 성과 차트 인터랙션
         - 시장 이벤트 타임라인 탐색
                ↓
         "관심 포인트 발견"
         - "내가 거래하는 BTC는?"
         - "2024년 약세장에서도 +23%?"
         - "SL 10%는 적절한가?"
```

**KPI**:
- Bounce Rate < 60%
- Demo 인터랙션율 > 40%
- 평균 체류 시간 > 2분

#### Stage 2: Exploration (탐색, 비회원)
**목표**: 5-10분 내 핵심 기능 체험

```
[데모 대시보드]
        ↓
 "전략 카탈로그 탐색"
 - BB Squeeze, HV Squeeze, ATR Breakout 등
 - 각 전략 핵심 지표 비교 (승률, PF, MDD)
        ↓
 "전략 상세 진입"
 - 2년 백테스트 결과 (월별 PnL)
 - 코인별 성과 Top 10 / Bottom 10
 - 주요 파라미터 설명 (SL, TP, 시간 필터)
        ↓
 "시장 컨텍스트 확인"
 - 2024년 5월: BTC -15% → 전략 -8% (방어 성공)
 - 2025년 1월: 알트시즌 → 전략 +31% (포착 성공)
        ↓
 "제한 노출"
 - "파라미터 변경하려면 회원가입"
 - "전체 코인 리스트 보려면 Pro"
```

**KPI**:
- 전략 상세 진입율 > 60%
- 시장 컨텍스트 클릭율 > 30%
- 회원가입 전환율 > 15%

#### Stage 3: Experimentation (실험, 무료 회원)
**목표**: 자신만의 전략 파라미터 찾기

```
[회원가입 완료]
        ↓
 "온보딩 튜토리얼"
 - 3단계 가이드 (전략 선택 → 파라미터 조정 → 결과 해석)
 - Interactive Tooltip 제공
        ↓
 "기본 파라미터 조정"
 - SL: 5% / 7% / 10% 비교
 - TP: 4% / 6% / 8% 비교
 - 시간 필터: 커스텀 (최대 3개 조합)
        ↓
 "즉시 시뮬레이션 실행"
 - 10초 내 결과 반환 (캐싱 활용)
 - Before/After 비교 테이블
 - "승률 +3%p, MDD -5%p" 강조
        ↓
 "결과 저장/공유"
 - 최대 3개 시뮬레이션 저장 (Free)
 - 링크 공유 기능 (바이럴 효과)
        ↓
 "Pro 티저"
 - "535코인 전체 보기 (Free: 50개만)"
 - "OOS 검증 결과 확인 (Pro 전용)"
 - "포트폴리오 시뮬레이션 (Pro 전용)"
```

**KPI**:
- 첫 파라미터 조정률 > 70%
- 평균 시뮬레이션 횟수 > 3회/세션
- 결과 공유율 > 10%
- Pro 전환율 > 8%

#### Stage 4: Conversion (전환, Pro 회원)
**목표**: 지속적 가치 제공으로 결제 유도

```
[Pro 전환 트리거]
 - Free 한계 도달 (시뮬레이션 3개 초과)
 - OOS 검증 필요성 인지
 - 커뮤니티 상위 전략 접근 욕구
        ↓
 "Pro 가입 (월 $19 / 연 $179)"
 - 7일 무료 체험 제공
 - 신용카드 정보 미리 등록 (자동 결제)
        ↓
 "Full Access 활성화"
 - 535코인 전체 백테스트
 - OOS 검증 결과 (2024/2025/2026 분리)
 - Monte Carlo 시뮬레이션 (1,000회)
 - 포트폴리오 상관관계 분석
 - 멀티 거래소 비교 (Binance/OKX/Bybit)
        ↓
 "고급 기능 사용"
 - 커스텀 코인 리스트 (블랙리스트)
 - 알림 설정 (시장 이벤트 발생 시)
 - API 키 연동 (조회 전용, Phase 3)
        ↓
 "결과 분석 심화"
 - 월별/연도별 드릴다운
 - 코인별 승률/PF 히트맵
 - 시장 레짐별 성과 (Bull/Bear/Sideways)
```

**KPI**:
- 7일 무료 체험 → 유료 전환율 > 40%
- 첫 달 Churn Rate < 15%
- 평균 세션 길이 > 15분

#### Stage 5: Habit (습관, 충성 고객)
**목표**: 주 1회+ 재방문, 연간 구독 유지

```
[정기 방문 루프]
 (주 1회)
        ↓
 "새 시장 이벤트 확인"
 - BTC ATH 돌파 → 전략 성과 어떤가?
 - 알트시즌 진입 → 포트폴리오 조정 필요?
        ↓
 "전략 성과 모니터링"
 - 실거래 계정 연동 (Phase 3)
 - 백테스트 vs 실전 괴리 분석
 - 주간 리포트 이메일 (자동 발송)
        ↓
 "커뮤니티 참여"
 - 인기 전략 파라미터 공유
 - 자신의 변형 전략 게시
 - 포럼 토론 (Reddit/Discord 연동)
        ↓
 "신규 전략 탐색"
 - PRUVIQ 팀이 추가한 새 전략
 - 커뮤니티 베스트 전략 (투표 기반)
        ↓
 "연간 구독 갱신"
 - 할인 혜택 (연간 결제 시 2개월 무료)
 - 충성 고객 배지/혜택
```

**KPI**:
- WAU (Weekly Active Users) > 40%
- 연간 구독 전환율 > 60%
- NPS (Net Promoter Score) > 50
- Churn Rate (3개월 후) < 10%

## 3. 기능 명세 (Feature Specification)

### 3.1 MVP (Phase 0-1): 핵심 검증

**출시 목표**: 2026년 Q2 (3개월)

#### F1. 전략 카탈로그
**목적**: 검증된 전략 목록 제공

| 요소 | 설명 | 우선순위 |
|------|------|---------|
| 목록 뷰 | 그리드/리스트 토글, 카드 형식 | P0 |
| 핵심 지표 | 승률, PF, MDD, Sharpe Ratio | P0 |
| 필터링 | 전략 타입(SHORT/LONG/DUAL), 승률 범위 | P1 |
| 정렬 | PF, 승률, 최근 업데이트순 | P1 |
| 검색 | 전략명, 키워드 (BB, Squeeze 등) | P2 |

**데이터 소스**: autotrader 백테스트 결과 (v1.0-v1.7)

#### F2. 전략 상세/결과 대시보드
**목적**: 백테스트 결과 시각화 + 이해

| 섹션 | 콘텐츠 | 차트 타입 |
|------|--------|---------|
| 개요 | 전략 설명, 핵심 파라미터 | 텍스트/태그 |
| 성과 요약 | 총 PnL, 승률, 거래 수, MDD | 숫자 카드 |
| 월별 PnL | 2024-2026 월별 수익 | 바 차트 (녹색/빨강) |
| 누적 수익 | 시작 $3,000 → 최종 자본 추이 | 라인 차트 |
| 코인별 성과 | Top 10 / Bottom 10 코인 | 테이블 + 미니 차트 |
| 청산 이유 | TP, SL, TIMEOUT 비율 | 도넛 차트 |
| 파라미터 | SL, TP, 시간 필터, 볼륨 필터 등 | 읽기 전용 (Free) |

**인터랙션**:
- 마우스 오버: 툴팁으로 상세 수치
- 클릭: 특정 코인/월 드릴다운
- CTA: "이 전략 커스텀하기" (회원가입 유도)

#### F3. 코인별 성과 비교
**목적**: 어떤 코인이 전략에 적합한지 발견

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 코인 리스트 | 535코인 전체 (Free: 50개만) | P0 |
| 정렬 | PnL, 승률, 거래 수, MDD | P0 |
| 필터 | 승률 > 60%, PnL > 0, 거래 수 > 10 | P1 |
| 개별 차트 | 코인별 월별 PnL 라인 차트 | P1 |
| 비교 모드 | 최대 3개 코인 성과 오버레이 | P2 |

**데이터 소스**: `data/ultimate_results/*.csv`

#### F4. 기본 시장 지표
**목적**: 전략 성과를 시장 맥락에서 이해

| 지표 | 출처 | 시각화 |
|------|------|--------|
| BTC 가격 | CoinGecko API | 라인 차트 (오버레이) |
| BTC 변동성 | CoinGecko | 영역 차트 (배경) |
| 주요 이벤트 | 수동 입력 (Phase 0) | 타임라인 마커 |

**예시 이벤트**:
- 2024-03: BTC 반감기
- 2024-11: 트럼프 당선
- 2025-01: BTC ETF 승인

#### F5. 회원가입/로그인
**목적**: 사용자 식별 + 데이터 저장

| 방식 | 우선순위 | 비고 |
|------|---------|------|
| 이메일/비밀번호 | P0 | 기본 방식 |
| Google OAuth | P1 | 전환율 향상 |
| GitHub OAuth | P2 | 개발자 타겟 |

**가입 시 제공**:
- 3개 시뮬레이션 저장 슬롯
- 기본 파라미터 조정 권한

### 3.2 Phase 2: 차별화 기능

**출시 목표**: 2026년 Q3 (6개월 후)

#### F6. 파라미터 커스텀
**목적**: 사용자 맞춤 전략 시뮬레이션

| 파라미터 | 조정 범위 | UI |
|---------|---------|-----|
| SL % | 3-15% | 슬라이더 |
| TP % | 3-20% | 슬라이더 |
| 시간 필터 | 24시간 중 선택 | 체크박스 (UTC) |
| 볼륨 필터 | 1.0-5.0x | 슬라이더 |
| 코인 리스트 | 커스텀 선택/제외 | 멀티셀렉트 (Pro) |

**시뮬레이션 실행**:
- "Run Simulation" 버튼 → 백엔드 큐 처리
- 평균 10-30초 소요 (캐싱 시 < 5초)
- 결과: Before/After 비교 테이블
  - 승률: 68.6% → 71.2% (+2.6%p) ✅
  - MDD: -33% → -28% (-5%p) ✅
  - 거래 수: 2898 → 2156 (-742) ⚠️

**제한**:
- Free: 시간 필터만 조정 가능 (SL/TP 고정)
- Pro: 모든 파라미터 조정 가능

#### F7. 멀티 거래소 비교
**목적**: 슬리피지/수수료 최적화

| 거래소 | 데이터 소스 | 비교 항목 |
|--------|------------|----------|
| Binance | 실거래 데이터 | 슬리피지, 수수료 |
| OKX | CCXT API | 유동성, 스프레드 |
| Bybit | CCXT API | 펀딩비 |

**시각화**:
- 거래소별 평균 슬리피지: 바 차트
- 거래소별 수수료 비교: 테이블
- 최적 거래소 추천: "Binance 권장 (슬리피지 -30%)"

## 6. 모바일 vs 웹 전략

### 6.1 플랫폼 우선순위

**결론: 웹 우선 (Desktop-first) + 반응형**

#### 근거
1. **타겟 사용 패턴**:
   - 백테스트 결과 분석: 큰 화면 필요 (차트, 테이블)
   - 파라미터 조정: 데스크톱에서 신중히 작업
   - 평균 세션 길이: 15분+ (모바일은 짧은 세션 선호)

2. **경쟁사 분석**:
   - QuantConnect: 웹 전용
   - TradingView: 웹 우선, 모바일 보조

3. **비용 효율**:
   - 네이티브 앱: 개발/유지보수 비용 2배
   - PWA (Progressive Web App): 반응형 웹으로 모바일 커버

### 6.2 모바일 전략

#### Phase 0-1 (MVP): 반응형 웹만
- Bootstrap 5 Grid System
- 모바일 뷰포트: 320px-768px 대응
- 핵심 기능만 최적화 (전략 탐색, 결과 보기)

#### Phase 2: PWA 전환
- Service Worker 추가 (오프라인 캐싱)
- "홈 화면에 추가" 프롬프트
- Push 알림 (시장 이벤트)

#### Phase 3: 네이티브 앱 검토
- MAU > 10,000 시 고려
- React Native 활용 (웹 코드 재사용)

### 6.3 모바일 핵심 기능

| 기능 | 모바일 우선순위 | 데스크톱 대비 |
|------|---------------|-------------|
| 전략 목록 탐색 | P0 | 동일 |
| 전략 결과 보기 | P0 | 단순화 (차트 1개) |
| 시장 컨텍스트 | P1 | 타임라인 세로 스크롤 |
| 파라미터 조정 | P2 | 슬라이더만 (고급 옵션 숨김) |
| 포트폴리오 시뮬 | P3 | 미지원 (데스크톱 권장) |

### 6.4 반응형 디자인 규칙

#### Breakpoints
```css
/* Mobile */
@media (max-width: 767px) {
  - 1열 레이아웃
  - 사이드바 → 햄버거 메뉴
  - 차트 높이 축소
}

/* Tablet */
@media (min-width: 768px) and (max-width: 1023px) {
  - 2열 레이아웃
  - 사이드바 → 아이콘 + 툴팁
}

/* Desktop */
@media (min-width: 1024px) {
  - 3열 레이아웃
  - 사이드바 전체 표시
}
```

#### 모바일 UX 최적화
1. **터치 타겟 크기**: 최소 44x44px (Apple HIG)
2. **폰트 크기**: 최소 16px (iOS 자동 줌 방지)
3. **차트 인터랙션**: 터치 제스처 (핀치 줌, 스와이프)
4. **로딩 최적화**: Lazy Load (차트는 스크롤 시 로드)

## 7. 성공 사례 참고

### 7.1 TradingView (차트 플랫폼)

**핵심 전략**:
- Freemium 모델: 무료로 기본 차트 → Pro로 고급 기능
- 커뮤니티: 사용자 제작 지표/전략 공유 → 네트워크 효과
- 반응형 웹: 모바일 앱도 제공하지만 웹이 주력

**PRUVIQ 적용**:
- ✅ Freemium 모델 동일
- ✅ 커뮤니티 Phase 3 계획
- ✅ 웹 우선 전략

### 7.2 QuantConnect (알고 트레이딩)

**핵심 전략**:
- 오픈소스 엔진 (LEAN): 투명성으로 신뢰 구축
- 교육 콘텐츠: Bootcamp, Docs 충실
- 기관급 데이터: 퀀트 전문가 타겟

**PRUVIQ 적용**:
- ✅ 오픈소스 엔진 (autotrader)
- ✅ Docs 페이지 계획
- ⚠️ 크립토 전문화 (주식 아님)

### 7.3 Stripe (핀테크 SaaS)

**핵심 전략**:
- 개발자 친화: API 문서 최고 수준
- 온보딩: 5분 내 첫 결제 테스트 가능
- 투명한 가격: 숨겨진 비용 없음

**PRUVIQ 적용**:
- ✅ 투명한 가격 (Pricing 페이지)
- ✅ 빠른 온보딩 (가입 없이 체험)
- Phase 3: API 문서 제공

### 7.4 Notion (생산성 SaaS)

**핵심 전략**:
- Product-Led Growth: 제품 자체가 마케팅
- 바이럴: 공유 기능으로 자연스러운 확산
- 템플릿: 사용자 제작 템플릿 갤러리

**PRUVIQ 적용**:
- ✅ 공유 기능 (Phase 2)
- ✅ 커뮤니티 전략 공유 (Phase 3)
- 🎯 "전략 템플릿" 개념 활용 가능
